# web-source-card 使用流程

`components/web-source-card.html` 是一张固定样式的"网络来源卡片"。这份文件讲清楚：**何时触发、如何搜索、如何填卡片**。

## 何时触发

扫描教案时，只要遇到以下任一类型就考虑用 web-source-card：

1. **最新数据 / 统计** —— "目前全球……"、"最新的……"、"近年来……"
2. **具体人物事件** —— 教案里点名的科学家、历史事件、具体公司/产品
3. **真实案例** —— "以 XX 为例"、"某地发生的……"
4. **地理/历史坐标** —— 具体地点、年份、里程碑
5. **可核验的量化事实** —— 海拔、速度、温度极值、经济数据

**不要触发的情况**：
- 纯概念定义（课本已讲清楚的）
- 纯原理推导（物理/数学公式）
- 已经写死在教案里的习题答案

每个课件建议控制在 2-4 张 web-source-card，太多会偏离教学主线。

## 工作流（四步）

### Step 1 · 构造查询

把教案里的表述转成 3-8 个词的搜索查询。**用中文查中文主题，用英文查国际话题**。

| 教案原文 | 查询示例 |
|---|---|
| "目前世界最高峰珠穆朗玛峰的海拔" | `珠穆朗玛峰 最新海拔 测量` |
| "近年全球平均气温变化" | `全球平均气温 2024 最新数据` |
| "屠呦呦因发现青蒿素获诺贝尔奖" | `屠呦呦 青蒿素 诺贝尔奖 年份` |

### Step 2 · 搜索并筛选

```
WebSearch(query="珠穆朗玛峰 最新海拔 测量")
```

从返回结果里挑 **1 个最权威**的：
- 优先：官方网站（.gov / .edu / 百科 / 权威媒体）
- 次选：维基百科、新华社、BBC、央视
- 避免：广告农场、不知名博客、日期过旧（> 2 年）

### Step 3 · 抓详情

```
WebFetch(url="<selected URL>", prompt="Extract: (1) the main fact/number related to <topic>, (2) 2-3 supporting facts, (3) the source name, (4) the publication date if available")
```

让 WebFetch 自己帮你抽结构化字段。如果第一个来源信息不全，再 fetch 第二个。

### Step 4 · 填入卡片

打开 `components/web-source-card.html`，用抓到的字段替换占位符：

```html
<div class="web-card">
  <span class="web-card-badge">📡 来自网络</span>
  <div class="web-card-title">珠穆朗玛峰最新测量高度 8848.86 米</div>
  <div class="web-card-body">
    <ul>
      <li>2020 年中尼联合测量公布的高度为 <b>8848.86 米</b></li>
      <li>比 1975 年中国测量数据高出 <b>0.13 米</b></li>
      <li>测量采用了 GNSS、重力、水准等多种手段</li>
    </ul>
  </div>
  <div class="web-card-source">
    <span>来源：<a href="https://www.xinhuanet.com/..." target="_blank" rel="noopener">新华社</a></span>
    <span>抓取于 2026-04-10</span>
  </div>
</div>
```

然后把整张卡片插到一个 `content-two-col` 的右栏，或者单独占一页（标题 + 这张卡片 + 一段过渡文字）。

## 质量红线

1. **URL 必须真实**—— 只能用 WebSearch/WebFetch 真正返回的 URL，禁止凭印象编 URL
2. **数字要对得上原文**—— 直接 quote 原文里的数值，不要约算或凑整
3. **日期要准确**—— 用实际抓取的日期，不要写"今天"或占位字符
4. **不超过 4 条要点**—— 超过就要拆成多张卡片或者改回用 info-card
5. **英文来源要翻译**—— 卡片里是中文课件，body 用中文改写，但保留英文原词（人名/术语）

## 失败处理

- **搜索无结果** → 改用更通用的查询词重试一次；再不行就跳过这个点，用普通 info-card 替代
- **WebFetch 失败** → 换另一个搜索结果；多个都失败就放弃 web-source-card
- **信息相互矛盾** → 优先取最新、最权威的那一条，在 body 里不要同时列冲突数据

## 示例：完整一轮

教案原文：
> 青藏高原是世界上最高的高原，对亚洲气候有巨大影响

触发原因：教案里说"最高的高原"但没给具体数据和影响范围——这正是 web-source-card 的用武之地。

操作序列：
1. `WebSearch("青藏高原 平均海拔 面积 气候影响")`
2. 选择结果里的"中国科学院青藏高原研究所"或维基百科条目
3. `WebFetch(url, "Extract elevation, area, and climate impact")`
4. 填入卡片：
   - 标题："青藏高原：世界屋脊的量级"
   - 要点：平均海拔 4000+ 米 / 面积约 250 万 km² / 影响亚洲季风系统
   - 来源：中国科学院青藏高原研究所 / 维基百科
5. 把这张卡片放进讲"地形对气候影响"那一页的右栏

这样整页就从"一句教科书式的陈述"升级成"带真实数据的案例卡"。
